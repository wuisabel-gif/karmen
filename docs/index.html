The real use case

Dispersion analysis

A rocket never flies exactly as designed: motor thrust varies batch to batch, fuel load and drag differ, wind shifts, and the launch angle is a hair off. Any one simulation tells you what happens for one set of those values. Kármán runs a million slightly-different rockets — a Monte Carlo sweep — and answers the questions that actually matter for a launch.

Will it reach the target?

e.g. 81% of flights cross the 100 km Kármán line.

How far will it land?

Downrange landing scatter that sizes the recovery & safety zone.

How risky is it?

Probability of structural (g-load) failure and of parachute failure.

What's the spread?

Apogee percentiles — P5 / P50 / P95 — not just an average.

Each rocket perturbs the nominal design — thrust ×[0.95, 1.05], fuel ×[0.98, 1.02], drag ×[0.90, 1.10], launch angle ±1° — so no two of the million flights are identical.

Under the hood

The GPU as a physics computer

No triangles, no pixels. A compute shader is really a parallel programming language — Kármán maps one rocket to one GPU thread and integrates a full trajectory in each.

1

Seed a million rockets

A per-thread PCG RNG scatters each rocket's motor, fuel, drag, and launch angle.

2

Integrate the flight

Semi-implicit Euler at 50 ms steps: thrust, exponential atmosphere, inverse-square gravity, transonic drag, wind.

3

Record the outcome

Apogee, peak speed, landing velocity, downrange, and status — explode on over-g, deploy a chute at apogee.

4

Reduce to statistics

A million outcomes collapse to percentiles, probabilities, and a report — in milliseconds.

// simulate.slang — one rocket, one thread
Result simulate(Rocket base, uint seed) {
  RNG rng = RNG(seed);
  Rocket r = perturb(base, rng);   // Monte Carlo

  for (int i=0; i<MAX_STEPS; ++i) {
    float m   = mass(r, t);
    float g   = gravity(py);
    float rho = density(py);
    // thrust · drag(wind, mach) · gravity
    integrate(dt);
    apogee = max(apogee, py);
    if (py <= 0.0) break;      // landed
  }
  return result(apogee, downrange, status);
}

Try it

Run it yourself

pip install slangpy numpy
python src/main.py spaceshot          # or falcon, student
python src/main.py falcon --count 250000
python src/main.py student --engine cpu

# outputs → report.txt, flights.csv, histogram.csv

On a machine with a GPU toolchain the Slang kernel runs the full million; otherwise a vectorized numpy engine runs the same physics on the CPU, so it works anywhere.
